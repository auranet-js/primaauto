# Historia wersji motywu `primaauto2026`

Standalone motyw produkcyjny zastępujący Hello Elementor + Elementor Pro. Źródłem prawdy jest `~/domains/primaauto.com.pl/public_html/wp-content/themes/primaauto2026/` — ten plik to kontekstowy changelog w repo.

| Data | Wersja | Uwagi |
|------|--------|-------|
| 2026-04-30 | 1.0.6 | **Z-index fix dla mobile menu nad sticky inventory toolbar.** `.pa-header` z z-index: 100 → 9000. Mobile menu (`.pa-mobile-menu` z z-index: 200) jako dziecko `.pa-header` było scope'd do stacking context z-index 100 — inventory sticky toolbar (`.aa-inv__mobile-bar` z z-index: 100, dalej w DOM) renderował się NAD hamburger menu. Po zmianie cały stacking context headera (włącznie z mobile menu) jest nad innymi sticky elementami (inventory toolbar 100, single sticky title 90). Inventory drawer (10001) i overlay (10000) celowo pozostają wyższe (gdy drawer otwarty user filtruje, nie przegląda menu). |
| 2026-04-30 | 1.0.5 | **Mobile padding + listing karuzela fix w hub'ach + footer mobile.** (a) **Hub container padding** — `.aa-container { padding: 0 12px }` (zamiast `padding: 0`) na `<=768px`. Treść hubów marek/modeli i `/marki/` nie skleja się z krawędzią ekranu. (b) **Karuzela "Najnowsze oferty" rozjechana** — root cause: `wp_kses()`/`wp_kses_post()` na merge'd `$wiki + $bar` strip'owało `<img fetchpriority>`/`decoding` i `<svg>` attrybuty, robiąc broken HTML z niedomkniętymi `</div>`/`</a>` → przeglądarka parsowała jako kaskadę zagnieżdżonych anchorów = "wysokie kolumny" pionowe zamiast horizontal slider. Fix w `taxonomy-make.php` + `taxonomy-serie.php`: split przez `{{LISTINGS_BAR}}` placeholder, sanityzowanie tylko user-side wiki content, `$bar` (nasz hub-listings HTML) wstawiany RAW. (c) **Defensive flex slider** — `.aa-hub__latest-wrap .aa-home__latest { display: flex !important; flex-direction: row !important; flex-wrap: nowrap !important }` w mobile media query, karty proporcjonalne do viewportu (`flex: 0 0 70vw; max-width: 230px; min-width: 180px`) zamiast fixed 210px, niższy obrazek (130px), section header `flex-wrap: wrap`. Scroll-snap added. (d) **Hub typografia mobile** — H1 26px (było 28px), h2 22px, h3 18px, USP padding zmniejszone, site-main padding-block 24/60px (było 40/80px), brand-grid gap 10px. (e) **Footer mobile** — brand col centered, social icons centered z flex-wrap, nav listy centered, krótszy padding. |

(c) **Submenu „Informacje" na mobilnym zwinięte** — CSS `.pa-mobile-menu__list .sub-menu { display: none }`, toggle przez `aria-expanded` na `<li>`. JS dodaje `<button class=pa-mobile-toggle>` z chevronem obok parent linku; klik rozwija submenu bez zamykania drawera (`stopPropagation`). Rodzic (link do `/informacje/`) działa normalnie. | 2026-04-24 | 1.0.2 | **Admin bar offset + bezpieczny fallback „Nawigacja" + link `/marki/`.** (a) Sticky `.pa-header` chował się pod pasek admina WP — dodane `body.admin-bar .pa-header { top: 32px }` + media query `<=782px` → `top: 46px` (WP admin bar grubszy na mobile). (b) Footer kolumna „Nawigacja" była pusta — theme_mod `nav_menu_locations` wskazywał `menu-2 → term 3273` który nie istnieje w DB (jedyne menu to term 6033 „Header" pod `menu-1`). `has_nav_menu('menu-2')` zwracał true, `wp_nav_menu()` nic nie renderował, fallback nie odpalał. Fix: `wp_nav_menu` z `echo=>false` + warunek na niepusty output → fallback gdy menu wskazuje na nieistniejący term. (c) Fallback rozszerzony z 4 do 6 linków: Samochody / **Marki** (nowy, `/marki/`) / W drodze / W Rzeszowie / **O nas** (nowy, `/informacje/o-nas/` bo `/o-nas/` = 301) / Kontakt. |
| 2026-04-24 | 1.0.1 | **Fix podwójnego headera.** `header.php` renderował dwa warianty `.pa-header__inner` (desktop + mobile) przełączane klasami utility `.desktop-only`/`.mobile-only` z `base.css`. Ale reguła `.pa-header__inner { display: flex }` w `header.css` (ładowany po `base.css` — zależność `['primaauto-base']`) miała tę samą specificity (0,1,0) i wygrywała kolejnością → `display: flex` nadpisywał `display: none` z `.mobile-only`, efekt: oba warianty renderowane jednocześnie na desktopie. Fix: `.pa-header__inner--desktop { display: flex }` + `.pa-header__inner--mobile { display: none }` z media query `@media (max-width: 768px)` odwracającą stan — selectory BEM same ustawiają visibility, utility klasy `.desktop-only`/`.mobile-only` usunięte z dwóch divów w `header.php` (zostaje tylko w `.pa-mobile-menu.mobile-only`, to osobna instancja). Bump w `style.css` + `PRIMAAUTO_THEME_VERSION` w `functions.php`. |
